# Activity Review Interface

> **Last Updated:** 2025-12-26
> **Parent:** [Navigation Index](2025-12-25-Navigation-Index.md)

---

## Purpose

This interface enables users to **review AI suggestions with full context**. Users see the information they need to make an informed decision before any accept/reject controls are offered.

This implements the "AI proposes, user disposes" philosophy correctly: the user can only dispose after seeing what they're disposing of.

---

## Design Principle

> **No action without context.**
>
> Every accept/reject decision must be preceded by showing the user:
> - What the activity was (screenshots, window titles, timestamps)
> - Why AI made this suggestion (reasoning/confidence)
> - What the alternatives are (if AI was uncertain)

---

## When It Appears

- User clicks "Review Now" from AI Transparency Summary
- User selects "Review Pending Items" from View menu
- User clicks on an activity flagged as "Needs Review"

---

## Key Characteristics

| Characteristic | Implementation |
|----------------|----------------|
| **Context first** | Screenshots and details shown before any action buttons |
| **AI reasoning visible** | User sees why AI made the suggestion |
| **One at a time** | Focus on single activity, not overwhelming list |
| **Split requires visual** | Split option only appears when viewing screenshot timeline |
| **Correction preserves context** | Screenshot remains visible when picking different bucket |

---

## Actions

| Action | When Available | Result |
|--------|----------------|--------|
| Accept | Always | Apply AI suggestion, move to next |
| Reject | Always | Mark as non-billable, move to next |
| Correct | Always | Show bucket picker alongside context |
| Split | Viewing screenshots | Show timeline with split point selector |
| Skip | Always | Leave for later, move to next |
| Exit | Always | Return to main view, remaining items stay pending |

---

## What This Interface Does NOT Do

- Does not show bulk checkboxes (each decision requires seeing context)
- Does not show action buttons without corresponding context
- Does not allow split without viewing screenshots
- Does not assume user knows what to do without showing them what to decide on
